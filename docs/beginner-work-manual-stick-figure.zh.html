灵台工作手册

你可以把灵台想成一个“长期 AI 助手”。它不只会聊天，还能记住项目、读文件、使用工具、找其他 AI 帮忙；如果外接渠道已配置且可用，通常会把结果发回你发起任务的地方。

1｜灵台到底是什么？

不是普通聊天框

普通聊天框像“你问一句，它答一句”。灵台更像一个在项目里长期工作的 AI 组织。

有自己的家

每个项目会有 .lingtai/。里面有智能体、信箱、记忆、日志、配置、运行状态和可视化记录。

能长大

任务复杂时，它能叫几个“临时小帮手 Daemon”一起查资料，也能培养“长期专门助手 Avatar”。

一句话：灵台不是只会陪你聊天的 AI，而是一个能长期帮你管理项目、记住事情、分配任务的 AI 工作系统。

2｜十分钟上手流程

安装并打开。README 当前推荐 Homebrew；第一次进项目后启动 TUI。如果你没用过命令行，请先找会电脑操作的人帮你完成安装；下面这些命令是输入到“终端”里的。
brew install lingtai-ai/lingtai/lingtai-tui
mkdir my-project && cd my-project
lingtai-tui
首次配置。跟着 TUI 选模型/配方/语言/工具。第一次不知道选什么，可以先选 Tutorial 或默认引导。
发一个小任务。例如：“帮我整理这个项目的文件结构，写一份 5 条以内的总结。”先用小任务确认它能读文件、能回复。
学 6 个常用命令。/setup 配模型和行为；/kanban 看助手和项目状态；/mcp 配微信/Telegram/邮箱等外接渠道；/skills 看技能；/viz 看组织图；/doctor 排障。
开始长期项目。告诉它项目背景、你的偏好、哪些内容要长期记住；重要产物让它写成文件或 HTML。

小提醒：普通用户请优先按 README 推荐的方法安装或升级。不要随便用 pip install lingtai，它可能不是你真正要升级的部分。大陆网络卡住时，看 README 里的清华镜像 / Gitee tap 说明。Windows 路线仍以最新 README/官网为准。

3｜灵台会做什么？

读文件
整理项目资料、写报告、做表格。
查网页
搜索、自动读取网页内容、核对资料来源。
看图片
识别截图、图表、海报文字。
用工具
在你允许时执行电脑命令、运行测试、生成网页，或请写代码工具帮忙。

外接渠道

配置相应 MCP/插件后，同一个长期助手可以从 TUI、微信、Telegram、飞书/Lark、WhatsApp、邮件等入口收到任务。配置正确且渠道可用时，通常回到原渠道。

可视化

/viz 或 Portal（图形化展示页）可以看见谁在线、谁给谁发信、组织怎样长大。

4｜灵台怎样“记住事情”？

对话

像课堂草稿纸，当前有用，太长会清理。

当前便签 Pad

像任务看板，记录临时笔记、当前计划和下一步。

项目知识 Knowledge

像项目档案，保存私有事实、路径、长期决策。

做事方法 Skills

像可复用教程，把做事方法沉淀给以后再用。

长期规则 Character

像性格和长期规则，告诉它“我是谁、我怎么做事”。

整理记忆 Molt / 凝蜕

聊天太长、上下文快装不下时，它会把重要内容整理成“给下一轮自己的交接信”，再轻装继续。像收拾书包，不是失忆。

给初学者的用法：重要背景要明确说“请记到长期知识/当前便签里”；复杂任务做完，可以说：“请把这次任务的背景、已完成内容、下一步计划整理成一段，下次我可以直接接着做。”如果你看到它提示“上下文很满/需要凝蜕”，就让它先整理现场再继续。

5｜怎样触发分神 / 分身？

临时小帮手 Daemon / 分神

像临时请 3 个同学分头查资料。适合：资料很多、内容很杂、需要交叉校对，但你只想要最后结论的任务。做完就把结论交回来，不长期存在。

长期专门助手 Avatar / 分身

像培养一个专门同事。适合：长期项目、固定领域、需要记住历史和风格的任务。分身会长期存在，越用越像一个固定岗位。

你可以这样叫分神

“这个资料很多，请派 3 个分神：一个查来源，一个看有没有错，一个把结果改成初中生能懂。”

你可以这样叫分身

“这个营养科普项目要长期做，请建一个专门负责证据核验的分身，以后固定找它。”

什么时候别乱开？

很小的事不用分神；需要真实发消息、删文件、发 issue、推代码的事，要先让人确认。分身适合长期岗位，不适合一次性小任务。

编码工具是“手”，灵台是“长期大脑”

如果你不写代码，可以先跳过这一段。Claude Code、Codex、OpenCode 等更适合精确改代码；灵台更适合计划、记忆、调度、沟通、长期项目管理。两者可以搭配。

6｜上下文“爆炸”了怎么办？

上下文可以理解成 AI 这一轮能摊在桌面上的纸。任务越长，纸越多；快放不下时，就要“收桌面”。灵台的办法叫 凝蜕 / molt：先把重点打包，再换一张干净桌子继续。

信号

对话很长、它提示 context/molt、开始反复找不到前文、或一个任务做了很多轮还没收束。

正确做法

让它先写交接：“请总结已完成、未完成、关键文件、下一步、要联系谁，再凝蜕/继续。”

不要这样

不要在快满时继续塞大量新任务；不要让它什么都不总结就刷新；不要把“凝蜕”当作失败，它是正常收纳。

一句话模板：“先收功：把当前任务、文件路径、结论、坑点、下一步整理好；该记入长期知识/便签的记好；然后再凝蜕继续。”

7｜常见 / 指令小抄

这些是在 LingTai TUI 里常见的斜杠命令。不同版本可能会增减，最终以 TUI 里实际显示和最新 README 为准。

/setup

第一次设置或重新检查项目、模型、渠道等基础配置。

/kanban

看当前有哪些 agent、任务、状态，像项目看板。

/mcp

配置微信、Telegram、飞书、邮箱等外接工具；没配好就不能从这些渠道收发。

/skills

查看/管理“做事方法手册”。复杂项目可以把流程写成 skill，避免每次从零开始。

/viz

打开可视化网络，看 agent 之间的关系和历史。

/insights

看系统整理出的洞察、回顾或运行观察。

/sleep

让 agent 休息；它仍可被消息唤醒。

/refresh

刷新当前 agent 的身体/配置。适合改配置后重载；不是随便清空一切。

/doctor

体检：排查启动、配置、工具、日志等问题。

/cpr

复苏卡住或假死的 agent。先诊断，再复苏。

/clear

强制重置某个 agent。影响较大，谨慎用。

/projects

切换或查看项目网络。

8｜心流是什么？

心流可以理解成“AI 在空闲时的自我复盘”。它会回看最近做过的事，提醒自己哪里可能遗漏、哪里应该沉淀、哪里该继续。它不是人发来的消息，也不是必须照做的命令。

像什么？

像一个人做完一天工作后写反思：“今天哪里做得好？哪里可能漏了？下次怎么更稳？”

有什么用？

帮助 agent 发现盲点：该不该回信、该不该写入知识、该不该做成 skill、是否需要休息或收功。

注意边界

心流只是内心提醒，不代表外界事实。涉及人类新消息、文件变化、网页资料时，仍要通过对应渠道核实。

9｜安全、隐私与排障

密钥不要外泄

API Key、token、密码可以理解成“软件钥匙”，只放受保护配置或 .secrets/。不要贴进公开报告、issue、截图。

外部动作要确认

发消息、提 issue、删文件、推代码都是真实动作。在你确认之前，不要让它自动发送、删除或发布内容。

专业内容要复核

医学、营养、法律、金融等内容，AI 可辅助整理，但最后要人类专家审核。陌生外部邮件也不应自动回复，除非用户确认。

常见问题

打不开 / 命令找不到
检查 Homebrew 的 bin 是否在 PATH；或者按 README 的安装详解重新装。
助手不回应
先等一小会儿：它可能在跑长工具、整理记忆或恢复。然后看 /kanban，再在 TUI 里用 /doctor；技术同学可再查 lingtai-tui list 和日志。
升级后没变化
重启 TUI；在 TUI 里用 /doctor 或在终端运行 lingtai-tui doctor。TUI 程序和 Python 运行时是两层。
技能/命令丢了
试 lingtai-tui bootstrap 或 TUI 里的 /doctor。如果切换模型/预设后微信、Telegram 等工具消失，检查 /setup、/mcp 和当前预设能力。

10｜一张小抄：怎么把任务说清楚

说目标：“帮我整理这周的会议记录。”
说范围：“只看 meeting-notes 文件夹，不要改原文。”
说形式：“输出 5 条重点和 3 个待办事项。”
说安全：“不要发给别人，只生成草稿；不要泄露 token 或密码。”
说回报方式：“完成后在微信告诉我摘要和文件路径。”

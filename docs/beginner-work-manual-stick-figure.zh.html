灵台不是只会聊天的网页机器人。它是本地工作台：TUI 负责界面，项目目录保存状态，agent 在里面读文件、跑命令、写报告、发消息。

1安装

普通用户安装的是 lingtai-tui，不要先去系统 Python 里 pip install lingtai。

brew install lingtai-ai/lingtai/lingtai-tui
lingtai-tui

2创建项目

一个主题建一个项目。项目里会出现 .lingtai/，agent 的信箱、日志、记忆都在这里。

3/setup

第一次启动先跑 /setup。不会选就用默认或 Tutorial 配置，先跑通再调细节。

4第一个任务

不要先问“你会什么”。直接给一个小任务，例如创建 notes.md。

安装最小路径

macOS 最推荐：先确认有 Homebrew，再装 LingTai。

brew --version
brew install lingtai-ai/lingtai/lingtai-tui
which lingtai-tui
lingtai-tui

若提示 command not found: brew，先装 Homebrew：
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

大陆网络卡住

先试默认安装。若卡在 Homebrew 索引，用清华 TUNA；若卡在 GitHub tap，用 Gitee tap；若 npm 证书报错，先清掉 npm 镜像变量。

出问题先 doctor

lingtai-tui doctor

TUI 里也可用 /doctor。它会检查 TUI、kernel、runtime 的关系。

看状态

/kanban/help/goal/mcp/viz

先记 /kanban 和 /doctor 即可。

怎样把任务说清楚

目标：我要做什么
材料：文件或链接在哪里
输出：要 Markdown、Word、HTML 还是代码
红线：不要发布、不要删除、先给我审核
不确定：先问我

说清楚这五项，agent 就少猜很多。

上下文满了

长任务做久后，让 agent 先收束现场：已完成、未完成、关键文件、下一步。需要时它会凝蜕，像把桌面收干净再继续。

文件 / knowledge / skill

一次性交付放文件；长期事实进 knowledge；以后还会复用的方法写成 skill。不要把所有东西都堆在聊天里。

主 agent / daemon / avatar

主 agent
判断与汇总
daemon
一次性脏活
avatar
长期专项

小白先不用主动开很多分身。大批量扫描、长测试、严厉审稿时，再让主 agent 派 daemon。

Issue #236 — README Development prerequisites

Conclusion: this docs-only change adds a concise prerequisites list to the top of the README ## Development section so contributors know which tools are needed before running the existing local validation commands.

What changed

The PR adds a ### Prerequisites subsection to README.md before the worktree and validation recipes. It does not modify runtime code, build scripts, dependency manifests, or contributor automation.

Prerequisite evidence

ToolWhy it is listed
GitThe README development flow uses a worktree-based contribution workflow.
Go 1.26.1 or newer compatible Go releaseBoth tui/go.mod and portal/go.mod declare go 1.26.1.
Node.js and npmThe portal web frontend under portal/web is built with npm ci and npm run build.
makeThe repository has component Makefiles in tui/ and portal/; the wording avoids implying a root-level make build.
POSIX-compatible shellThe README and installer use shell snippets; Windows users should use WSL/MSYS2 or equivalent.

Risk assessment

Low risk. The change is documentation-only. The Node.js wording intentionally says “recent LTS” instead of claiming the repository pins an exact Node version, because portal/web/package.json currently has no engines field.

Validation

  • grep -n '^go ' tui/go.mod portal/go.mod confirms both modules declare go 1.26.1.
  • test -f portal/web/package.json confirms the portal frontend manifest exists.
  • git diff --check passes.

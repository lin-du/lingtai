AED max attempts exposed in TUI /setup

Conclusion: the setup/runtime page now includes an advanced AED max attempts field. The value is written to init.json as manifest.max_aed_attempts, matching the kernel key used by AED recovery.

Behavior

  • Adds AED max attempts next to existing runtime controls such as stamina, context limit, soul delay, molt pressure, and max RPM.
  • Persists the setting through preset.AgentOpts.MaxAedAttempts into manifest.max_aed_attempts.
  • Hydrates existing agents in /setup from saved manifest.max_aed_attempts, so re-opening setup shows the current value.
  • Validation clamps values to 1..100; empty/invalid/zero values fall back to 3, matching the kernel's effective _read_init fallback.
  • Help text warns that AED is auto-error-recovery and that high values can retry for a long time.

Files changed

  • tui/internal/preset/preset.go — adds MaxAedAttempts, constants, clamp helper, and init.json write.
  • tui/internal/preset/preset_aed_attempts_test.go — covers defaults, clamping, and generated manifest output.
  • tui/internal/tui/firstrun.go — adds the runtime field to first-run and /setup, including hydration and save path.
  • tui/i18n/{en,zh,wen}.json — adds labels and help text in all three locales.

Notes

Kernel note:

The TUI first-run/setup default is now 5, so newly generated init.json manifests persist manifest.max_aed_attempts: 5. Empty, invalid, or zero input falls back to the same TUI default before writing.

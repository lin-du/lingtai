LingTai TUI PR explainer

Default agent context window moves from 200K to 300K

Jason asked to raise the LingTai TUI default context window now that Codex GPT-5.5 via OAuth/Codex is understood to support a larger practical window. This PR changes the default for newly generated agents while leaving existing user configs and provider-specific presets alone.

Before
200,000
Default context_limit for new TUI-generated agents.
After
300,000
New default with safer headroom below Codex GPT-5.5's reported 400K OAuth/Codex limit.
Scope
2 files
Preset generation default + first-run/setup UI fallback/display default.

What changed

  • tui/internal/preset/preset.go: DefaultAgentOpts().ContextLimit now returns 300000.
  • tui/internal/tui/firstrun.go: first-run/setup defaults and invalid-input fallbacks now use 300000.
  • No provider-specific preset limits were changed.
  • No existing project init.json files are mutated by this PR; this only changes newly generated/default values.

Why 300K?

OAuth/Codex GPT-5.5 public discussion points to a 400K Codex context window, while LingTai's current generated-agent default was 200K. A 300K default gives agents more usable room without treating the 400K provider ceiling as a safe operating target.

Validation

Focused validation for preset/default generation and first-run code should pass before merge:

git diff --check
(cd tui && go test ./internal/preset/...)
(cd tui && go test ./internal/tui -run 'FirstRun|Generate|Context' -count=1)

If broad TUI tests are run, remove transient tui/internal/tui/logs/ before final clean-status checks.

Risk notes

  • Low runtime risk: this is a default-value change, not a runtime scheduler or tool-call change.
  • Compatibility: smaller-context providers can still be configured with explicit preset values; the PR does not force existing agents upward.
  • Operator path: Jason separately asked local bots to update their own init.json values and refresh; that is an operational action outside this PR.

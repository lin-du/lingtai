TUI /notification command

Status: implemented and validated locally.

What changed

  • Added a slash command /notification to the palette and help assets.
  • Added NotificationModel, a read-only MarkdownViewer page scoped to the current agent.
  • The page shows an aggregate raw notification block from <agent>/.notification/*.json, plus one entry per channel file.
  • Press r in the view to reload current filesystem state.
  • Added English, Chinese, and Wenyan i18n strings and slash-command help entries.
  • Updated tui/internal/tui/ANATOMY.md.

Why

This gives users a direct way to see the notification block an agent currently sees, without relying on synthesized diary/text rendering. It complements the kernel-side notification-sync change: consumed notifications may remain in history as structured tool-call/tool-result records, while pending notifications are visible here as raw files.

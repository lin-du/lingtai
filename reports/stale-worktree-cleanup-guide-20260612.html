Stale Worktree Cleanup Guide

Docs PR · branch docs/stale-worktree-cleanup-guide-20260612 · 2026-06-12 · requested by Jason

TL;DR

The dev guide's worktree loop said "clean up the worktree" but never explained how, so stale worktrees accumulate across agents and repos. This PR adds a "Worktree hygiene" section to reference/contributing/SKILL.md — audit recipe, removal criteria, keep/skip list, removal commands, record-and-report step — and routes to it from the top-level lingtai-dev-guide router. Docs only; no code changes.

Baseline

An actual cleanup on 2026-06-12 swept secondary worktrees in Lingtai-AI/lingtai and Lingtai-AI/lingtai-kernel. The policy that emerged (merged-only, remote-gone-only, skip source-dirty, protect the live release target, force-remove only generated dirt) existed nowhere in the dev guide. This PR encodes it so future agents don't re-derive it — or get it wrong.

The new section's shape:

  1. Audit first — git fetch --prune, then a shell loop over git worktree list --porcelain printing per-worktree branch / merged / remote / dirty_files. Dry-run only; inspect dirty trees by hand.
  2. Removal criteria (all must hold) — secondary worktree; HEAD is ancestor of origin/main; remote branch gone or detached HEAD; clean or generated-only dirt (logs/, artifacts/, review-tool outputs).
  3. Keep/skip list — when in doubt, skip: primary checkout; live release/binary-target worktrees (e.g. release-vX.Y.Z-<date>); unmerged branches; remote branch still exists (likely open PR); source-dirty even if merged; other agents' worktrees unless explicitly in scope.
  4. Remove — git worktree remove (--force only for generated-only dirt), git worktree prune, git branch -d (never escalate to -D; a refusal means re-check the audit).
  5. Record and report — log removed paths/branches/SHAs and skipped+reason, save a report file, tell the human. SHAs are the recovery path.

Risks / decisions

  • Placed the procedure in contributing/SKILL.md (not gotchas) because it is a workflow, not a footgun; the gotchas file only mentions worktrees in passing.
  • The recipe deliberately fails safe: every ambiguous state (dirty, unmerged, remote exists, detached-but-protected) lands on "skip". The only force path is generated-only dirt.
  • Real cleanup-report paths from the 2026-06-12 sweep are referenced generically ("save a report file") rather than embedding private absolute paths.

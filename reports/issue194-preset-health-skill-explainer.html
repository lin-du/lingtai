Issue #194 — Swiss-knife preset health-check subskill

Conclusion: this change adds a documentation-only, read-only swiss-knife nested reference, preset-health, that gives agents a repeatable procedure to enumerate saved presets, classify each failure, and report actionable fixes without printing or mutating any secret. The swiss-knife router catalog and routing table are updated so the new reference is discoverable, and the embedded-skill extraction test is extended to cover the new file.

Why this was needed

Agents could call system(action="presets") and eyeball connectivity statuses, but there was no documented, repeatable workflow for checking all saved presets, classifying failures (expired key, missing credentials, unreachable endpoint, invalid model/config, connectivity failure), and reporting them safely. The triggering case: gemini-test.json reported connectivity.status = no_credentials with GEMINI_API_KEY not set, and a separate preset was expired — both required ad-hoc manual interpretation.

The classification taxonomy

The skill maps each preset to a fixed set of classes, grounded in the connectivity outcomes the TUI's /doctor probe already distinguishes (probeOK, probeAuthError, probeNoKey, probeNetworkError, probeRateLimit, probeOverloaded, probeEmptyResponse, probeOAuth):

ClassTrigger
okreachable, authed, valid model
no_credentialsapi_key_env declared but the variable is empty/unset
auth_failed / expired_keycredential present but rejected (401/403)
endpoint_unreachableDNS/connect/TLS/timeout to base_url
model_not_foundauth ok but model rejected/unknown
config_errormalformed JSON, missing provider/model, invalid base_url
rate_limitedreachable + authed but throttled (transient — retry)
connectivity_failurereachable but empty/invalid response envelope
unknownnone of the above

Secret safety

No secrets, ever. The skill checks credential presence only (set vs not-set), never value; refers to credentials by env-var name (e.g. GEMINI_API_KEY); and ships a redactor that strips bearer tokens, key-like strings, and URL query parameters from any raw error text before it lands in a report.

Read-only contract

The reference never edits ~/.lingtai-tui/presets/saved/*.json, agent init.json, or any manifest.preset.allowed list, and it does not switch the active preset to probe an inactive one. Remediation (set an env var, rotate an expired key, remove a preset from an agent's allowed list, update a retired model) is presented as a human-confirmed recommendation — the workflow stops at the report.

Risk assessment

Low risk. The change is documentation/skill-workflow plus one test-assertion line and a router catalog edit. No runtime Go logic, build scripts, or dependency manifests are touched. No script is added, so there is no executable surface to audit beyond the read-only shell snippets embedded in the SKILL.md.

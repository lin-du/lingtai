Fix Tutorial English language drift

Tutorial setup now follows the selected TUI language, and the English Tutorial prompt/curriculum no longer contains Chinese-language cues that could steer an English guide into Chinese.

Root cause

There were two language-leak paths. First, FirstRunModel.enterAgentNameDir initialized the new agent language from the selected preset's manifest.language before consulting the current TUI language. Second — the root cause Jason pointed at — the English Tutorial comment and lesson assets themselves contained Chinese identity/language cues (现代汉语, 文言, 菩提祖师, and Chinese metaphor glosses). Even when the UI and agent language were both English, those cues could nudge the model into Chinese.

Change

Fresh first-run / rehydration
Default agent language now follows tui_config.json language first.
Preset fallback
Preset manifest.language is still used if the TUI config language is empty or invalid.
/setup preservation
Existing agents still surface their saved init.json language when rerunning /setup.
English prompt hygiene
English Tutorial comment and curriculum assets now avoid CJK prompt cues; zh/wen localized files keep Chinese guidance.

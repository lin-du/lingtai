LingTai TUI PR explainer

Raise normal agent loop budget defaults to max_turns: 500

Prepared 2026-06-12 for PR #317. This is the separate follow-up PR Jason requested after the loop-guard investigation.

TL;DR

The TUI-created agent default now writes manifest.max_turns = 500 instead of 100. The shipped init templates/examples now describe max_turns as the per-wake LLM/tool-call loop budget, because tool-heavy work consumes the same guard budget.

Baseline

Observed problem
Long review/polling sessions can hit tool_loop_limit because the normal agent guard budget is only 100 for TUI-created agents.
Root cause
max_turns is currently used as a per-wake loop/tool-call budget, not merely an LLM request count.
Requested decision
Jason asked to be generous and set the default directly to 500.

What changed

tui/internal/preset/preset.gotemplates/init.jsoncexamples/init.jsoncsetup-playground.shpreset_test.go

// TUI-generated init.json default
manifest["max_turns"] = 500
// Per-wake loop budget: max iterations of the LLM/tool-call loop per
// wake cycle. Every tool call counts as a turn, not just LLM requests.
// Safety limit — agent sleeps once the budget is exhausted.
"max_turns": 500

The playground setup script now matches the wizard default. The preset test now pins generated manifest.max_turns == 500 so the default cannot drift silently.

Validation

  • Passed: cd tui && go test ./internal/preset -count=1
  • Passed: git diff --check
  • Reviewed all repo max_turns mentions; daemon override docs and old kernel design notes were left out of scope.

Risks and decisions

  • This does not remove the loop guard; it only raises the normal TUI-created agent default.
  • This does not change daemon-specific budgets, duplicate-call protection, invalid-tool handling, or polling behavior.
  • Existing agents keep their current init.json value until manually changed; this PR affects newly generated configs/templates/playgrounds.

Next steps

Review the small diff, then merge only with explicit approval. No release/tag work is included in this PR.

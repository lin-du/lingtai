LingTai adaptive recipe

Recommend IM connection in the first welcome

This change updates the bundled adaptive discovery recipe so the first-session greeting points users toward Telegram/Feishu/WeChat-style IM channels as the best interaction path.

Why

Jason noted that LingTai feels best when humans interact with agents through an IM channel: the agent can continue asynchronously, send proactive updates, and reach the human on mobile even after the TUI is closed.

What changed

  • Adaptive greet instructions now explicitly recommend connecting an IM channel early.
  • The recommendation explains the benefit: phone access, async replies, and proactive updates outside the TUI.
  • Adaptive comment playbook now tells agents to follow up once in the first session if no IM channel is connected.
  • All bundled adaptive locale variants were updated: default, zh, and wen.
  • recipe.json description now mentions early IM recommendation.

/mcp status wording

The wording is careful not to describe the command as the configuration mechanism. It says /mcp is for checking that addon configuration/connectivity succeeded after setup. Configuration itself is handled outside that status view.

Focused validation should pass with git diff --check and cd tui && go test ./internal/preset -count=1. The new test guards that adaptive recipe assets mention IM/Telegram//mcp, do not mention retired /addon, and do not describe /mcp as the configuration mechanism.

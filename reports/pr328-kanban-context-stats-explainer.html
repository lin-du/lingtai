What changed

  • tui/internal/fs/agent.go: added ContextStats, ContextToolCount, and ReadContextStats(dir). The parser reads history/chat_history.jsonl and counts retained entries, system/assistant/user rows, text input/output blocks, tool calls/results, and per-tool distribution.
  • tui/internal/tui/props.go: Ctrl+D detail now loads context stats and renders a “Current context statistics” section before recent activity.
  • Recent activity now keeps/render last 40 token-ledger rows instead of 20.
  • tui/i18n/{en,zh,wen}.json: updated recent-activity label and added the context-stats section title for all three locales.
  • ANATOMY.md files were updated because cited line numbers and PropsModel responsibilities changed.

Design notes

  • The source of truth is <agent>/history/chat_history.jsonl, matching TUI’s filesystem-only architecture: the kernel writes files; TUI reads them.
  • The stats are structural counts, not token attribution. Token budget still comes from .status.json and token ledgers.
  • The parser is best-effort: missing/unreadable history returns zero stats; malformed lines are skipped; legacy string content, top-level legacy text, and text_input/text_output block names are tolerated.
  • Per-tool rows are sorted by descending calls, then name, for deterministic display.

ctrl+o API-call spacing hotfix

Conclusion: the hotfix groups all ctrl+o verbose LLM-round-trip entries by api_call_id, so diary/internal output from separate API calls gets a blank separator while entries from the same API call stay together.

What broke

PR #323 fixed text_output spacing by API call, but thinking, diary, and text_input still did not inherit the derived api_call_id. That left diary/internal output visually glued together even though the hidden llm_response markers already identified API-call boundaries.

What changed

  • buildMessages now assigns derived API-call ids to thinking, diary, text_input, text_output, tool_call, and tool_result.
  • renderMessages now tracks one previous visible API-grouped verbose entry across internal output and tools.
  • apiCallGroupSeparatorBefore inserts a blank line only when the non-empty api_call_id changes.
  • Legacy tool streams without metadata keep the old tool_result → tool_call fallback separator.

Validation

  • Focused render/build tests for diary, mixed verbose entries, existing text output, and tool grouping.
  • cd tui && go test ./internal/tui -count=1
  • git diff --check

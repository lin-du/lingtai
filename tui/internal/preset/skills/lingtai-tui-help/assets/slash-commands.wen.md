# 斜杠之命

凡 `lingtai-tui` 中，闲谈之外诸事，皆由斜杠之命驱之：启视图、理器灵之生死、设运行之制。

## 用命令之面板

- 于讯框中入 `/`，则**命令面板**启。
- 续输以滤之，许模糊相合：`/skl` 合 `skills`，`/ref` 合 `refresh`。
- `↑`/`↓` 移其选，`Enter` 行所明之命。
- 于讯框中，`Enter` 发送；`Shift+Enter` 或 `Ctrl+J` 入换行。
- 有命于名后系参，如 `/sleep all`、`/refresh mimo`。

## 阅此帮助

`/help` 启此册于 Markdown 阅器。按 `Tab` 移焦于内容之栏，则 `PgUp`/`PgDn` 可翻页；
`Ctrl+E` 导今页于 `~/Downloads`；按 `Esc` 或 `q` 还邮视。

## 诸命概览

### 与器灵言
- `/btw` — 进一旁问，不扰器灵当下之务。
- `/insights` — 即索当下之务二三确切之见。
- `/goal` — 令当下器灵导人立或修一 active goal。

### 器灵之生死
- `/sleep` — 令器灵眠（可以 `/cpr` 醒之）。
- `/suspend` — 全冻器灵之程（可以 `/cpr` 醒之）。
- `/cpr` — 复挂起或已死之器灵。
- `/refresh` — 硬重启：自盘重载其设，可择换预设。
- `/clear` — 清对话之窗，存身份、便笺、codex。
- `/molt` — 即令器灵蜕（先存上下文，再重其窗）。
- `/nirvana` — 尽除诸物，自零而始。不可逆。

### 察器灵
- `/kanban` — 网络之盘，览各器灵之设与上下文之耗。
- `/skills` — 阅器灵之技目。
- `/knowledge` — 阅器灵之私知（别名：`/library`、`/codex`）。
- `/system` — 阅器灵之系统诸文（system.md、covenant 等）。
- `/daemons` — 察各器灵守护之运及其录。
- `/notification` — 观当前 Agent 近日所受告块，并可以左右翻阅。
- `/presets` — 览此器灵可以 `/refresh` 换之预设。

### 网络与分享
- `/mailbox` — 阅邮箱诸讯与附件。
- `/projects` — 阅已注之项目及其网络。
- `/export` — 导可复用之方以分享。
- `/viz` — 于浏览器启网络之图。

### 设置与诊断
- `/setup` — 器灵设置之导（供者、模型、诸能）。
- `/settings` — TUI 之偏好（主题、页幅、语言）。
- `/mcp` — 外服桥接之 MCP 控制台。
- `/doctor` — 诊连接之患。
- `/update` — 更 Python 核（须允）。
- `/login` — 察并理已存之凭。

### 此视与退出
- `/help` — 启此帮助之册。
- `/quit` — 退 `lingtai-tui`（器灵仍运于后台）。

## 诸命详解

### `/btw` — 向器灵进旁问
**用：** `/btw <尔之问>`

以尔问作一*洞见之询*而投之。器灵反思而答，不扰其当下所为——使尔于不令其离辙之际，探其
思绪。凡欲问与当务无涉之旁事，又不愿如常讯之扰其专者，宜用之。

### `/insights` — 即索洞见
**用：** `/insights`

令器灵即观当务，出二三确切之见。凡欲即得器灵所见之结构化读，而不待自动洞见之节（于
`/settings` 中启闭）者用之。

### `/goal` — 导立 active goal
**用：** `/goal` · `/goal <愿之草稿>`

投一 `goal.request` 事入当下器灵之 `.notification/system.json`。器灵当读
goal manual，与人同明 objective、success criteria、reminder delay 与取消之义；待人
确认诸目后，方可写 `.notification/goal.json`。`/goal` 自身不直造 goal 文卷。后若
dismiss `goal.reminder`，但隐其提醒；真欲取消 goal，当删 `.notification/goal.json`，
或置其 status 为 inactive/cancelled/done。

### `/sleep` — 令器灵眠
**用：** `/sleep`（当下器灵）· `/sleep all`（项目内诸器灵）

眠者，存其全状而暂止器灵，后可以 `/cpr` 醒之。凡欲止器灵耗资，而拟原地复之者用之。若需
对系统之程作更硬之冻，则用 `/suspend`。

### `/suspend` — 挂器灵
**用：** `/suspend`（当下器灵）· `/suspend all`（项目内诸器灵）

全冻器灵之程，须先 `/cpr` 复之方可续。凡欲对底层之程作硬止者用之——重于 `/sleep`。若但欲
可复之暂止，则用 `/sleep`。

### `/cpr` — 复挂起或已死之器灵
**用：** `/cpr`（当下器灵）· `/cpr all`（项目内诸器灵）

令挂起或已死之器灵复运。用以复 `/sleep`、`/suspend` 所止之器灵，或救已死者。常重启宜优用
`/refresh`——`/cpr` 专为已宕之器灵设。

### `/refresh` — 硬重启器灵
**用：** `/refresh`（重启，还默认预设）· `/refresh <名>`（换预设，如 `/refresh mimo`）·
`/refresh all`（诸器灵）

自盘重载 `init.json`、诸能与全设，乃重启器灵。系预设之名者，重启前先换当下预设；其预设须
在器灵 `manifest.preset.allowed` 之列（见 `/presets`）。此乃改设之后首选之命，亦用以换异之
LLM/能力预设——唯器灵确已宕时方用 `/cpr`。

### `/clear` — 清上下文之窗而重启
**用：** `/clear`

清器灵全部上下文之窗，以空白对话重启之。身份、便笺、codex 皆存——唯实时对话被除。凡对话
积噪或离向、尔欲洁净之始而不失持久之忆者用之。若欲先存上下文，则改用 `/molt`。

### `/molt` — 即令器灵蜕
**用：** `/molt`

强器灵即蜕：先存上下文，再重对话之窗。凡上下文之窗将满、又不愿失器灵所积者用之。异于
`/clear`，蜕者重置之前先存上下文。

### `/nirvana` — 尽除诸物，自零而始
**用：** `/nirvana`

除 `.lingtai/` 及器灵全部之数，令项目还于全新之态。**⚠️ 不可逆**，无从撤之。若但欲洁净之
对话，用 `/clear`；若欲存上下文而重置，用 `/molt`。唯尔确欲项目自零重来时方用 `/nirvana`。

### `/kanban` — 启器灵网络之盘
**用：** `/kanban`

示网络中各器灵之盘：其属、LLM 之设、诸能，及上下文之耗，一览而尽。凡欲于单一终端之视中
较诸器灵之设与其上下文窗之满者用之。若需图形化、基于浏览器之视，则用 `/viz`。

### `/skills` — 阅技目
**用：** `/skills`

启技目——器灵可按需召之可复用诸程。`Ctrl+T` 于器灵间切；于某技按 `Enter` 可入其夹中之文。
用以观器灵所能按需为者，或察某具体之技。

### `/knowledge` — 阅器灵之私知
**别名：** `/knowledge`、`/library`、`/codex` · **用：** `/knowledge`

阅器灵之私知——本地之决、笔记、参考及所迁之 codex 条，含其积研究与所得之持久知库。`Ctrl+T`
于器灵间切。用以读器灵为己所录，别于共享之技目（`/skills`）或系统诸文（`/system`）。

### `/system` — 阅器灵之系统诸文
**用：** `/system`

阅器灵之系统诸文：`system.md`、covenant、principle、procedures、pad 及 llm 之设。`Ctrl+T`
于器灵间切。用以读定义一器灵之持久骨架——其治理之文与运行之设——而非其对话或私知
（`/knowledge`）。

### `/daemons` — 阅守护之运
**用：** `/daemons`

启守护之阅器：察各器灵守护之运及其态、全任务、全 `chat_history` 之交，及全工具/事件之录。
用以溯某后台守护究竟何为。

### `/notification` — 观近日告块
**用法：** `/notification`

显当前所聚 Agent 近日所受之 agent-visible 告块（自 `logs/log.sqlite` 重建）。凡徽标有疑、Agent 言有告，或欲验近来实际注入彼灵之告块，可用之。左/右翻阅近告，`r` 重载，Esc/`q` 返聊天；此视只读，不 dismiss，亦不改告。

### `/presets` — 启预设之库
**用：** `/presets`

启限于当下器灵之预设库，唯示其 `manifest.preset.allowed` 列中之预设——即尔可以
`/refresh <名>` 换之者。当下所活之预设以 ● 标。尔可览各预设之 LLM 与诸能，并为其加一至五星
之费/质等（愈高愈善）；标会传于器灵，导守护/分身之择。此视乃只读之察加标签之编——预设之全
创建在 `/setup` 中行。

### `/mailbox` — 阅邮箱诸讯
**用：** `/mailbox`

阅每一邮箱之讯——收、发与 IMAP——含全文及内联附件之渲。凡欲得某器灵之全邮录（含外部 IMAP
邮与附件），而非实时对话之流者用之。

### `/projects` — 阅已注之项目
**用：** `/projects`

列已注之项目及其器灵网络。凡尔跨多 LingTai 项目而作、欲于一处览之及其网络者用之。

### `/export` — 导可复用之方
**用：** `/export` · `/export recipe`（显式之形）

导当下之设为可复用之方以分享。用以包尔器灵之设以复用与分享。

### `/viz` — 启网络之图
**用：** `/viz`

于浏览器启器灵网络之图——拓扑、邮流与器灵之态。须 `lingtai-portal` 在 `PATH` 中。用以实时
图形化观网络中器灵如何相关相通。若但需终端之概，则用 `/kanban`。

### `/setup` — 行器灵设置之导
**用：** `/setup`

启器灵设置之导，可改 LLM 之供者、模型、诸能与运行之设；其改传于诸器灵。用以首次之设，或对
供者、模型、诸能之任何实质之改。唯 TUI 之偏好（主题、语言）用 `/settings`；预设之全创建亦在
此导中行。

### `/settings` — 编 TUI 之偏好
**用：** `/settings`

启 TUI 之偏好：主题、邮件页幅、语言，及自动洞见之启闭。用以 TUI 自身之观与客户之行。若欲改
器灵之供者、模型或诸能，则用 `/setup`。

### `/mcp` — 启 MCP 控制台
**用：** `/mcp`

启 MCP 控制台，览各 MCP 桥之资、态与设——此诸桥（IMAP 邮、Telegram、飞书、微信）连器灵于
外服。凡设或排外服之集成者用之。改 MCP 之设后，行 `/refresh` 以使之效。

### `/doctor` — 诊连接之患
**用：** `/doctor`

对器灵之通连行诊——查 API 之钥、模型之可用与网络之设——并报所察。凡器灵不能启或止应、尔疑
凭、模型或网络之患者用之。若 `/doctor` 指凭之患，可配 `/login` 重认。

### `/update` — 更 Python 核
**用：** `/update`

较已安之 Python `lingtai` 核与 PyPI 之新（但览而不改），示 `今 → 新`，凡有所为，必先俟尔之允。允而后独以 uv/pip 升其核——不行 Homebrew 之 TUI 升，不重引预设，亦不触文件之搜。若察可编之开发安，则置而不动。尔但欲专更其核、而不欲牵 `/doctor` 诸余之效者用之。

### `/login` — 察并理凭
**用：** `/login`

示一切已存之凭之认证之态，并许必要时重认。凡某供者之凭已过或须重登者用之——多为应 `/doctor`
所暴之患。

### `/help` — 启此帮助之册
**用：** `/help`

启尔今所阅之帮助之册：斜杠之命如何运之概，及每命之参考条。按 `Esc` 或 `q` 还邮视。凡欲查某命
之用，或发尔未曾用之命者，任时用之。

### `/quit` — 退 lingtai-tui
**用：** `/quit`

退 `lingtai-tui`。除非尔先挂器灵，否则其于后台续运。若欲器灵亦止，退前用 `/suspend all`
（或 `/sleep all`）。

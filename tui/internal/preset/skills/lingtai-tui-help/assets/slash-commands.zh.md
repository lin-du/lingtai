# 斜杠命令

斜杠命令驱动 `lingtai-tui` 中除普通对话之外的一切：打开视图、管理智能体生命周期、配置运行时。

## 使用命令面板

- 在消息框中输入 `/` 打开**命令面板**。
- 继续输入即可过滤，支持模糊匹配：`/skl` 匹配 `skills`，`/ref` 匹配 `refresh`。
- `↑`/`↓` 移动选择，`Enter` 运行高亮的命令。
- 在消息框中，`Enter` 发送；`Shift+Enter` 或 `Ctrl+J` 插入换行。
- 部分命令在名称后接参数，例如 `/sleep all` 或 `/refresh mimo`。

## 浏览本帮助

`/help` 在 Markdown 阅读器中打开本指南。按 `Tab` 把焦点移到内容面板，便可用
`PgUp`/`PgDn` 翻页；`Ctrl+E` 将当前页导出到 `~/Downloads`；按 `Esc` 或 `q` 返回邮件视图。

## 命令一览

### 与智能体对话
- `/btw` — 提一个旁问，不打断智能体当前的工作。
- `/insights` — 立即索取关于当前任务的 2–3 条具体观察。
- `/goal` — 让当前 Agent 引导创建或修订一个 active goal。

### 智能体生命周期
- `/sleep` — 让智能体休眠（可用 `/cpr` 唤醒）。
- `/suspend` — 完全冻结智能体进程（可用 `/cpr` 唤醒）。
- `/cpr` — 复活已挂起或已死亡的智能体。
- `/refresh` — 硬重启：从磁盘重新加载配置，可选切换预设。
- `/clear` — 清空对话窗口，保留身份/便笺/codex。
- `/molt` — 立即让智能体蜕变（保存上下文后重置窗口）。
- `/nirvana` — 清除一切，从零开始。不可逆。

### 检视智能体
- `/kanban` — 网络面板，查看每个智能体的配置与上下文用量。
- `/skills` — 浏览智能体的技能目录。
- `/knowledge` — 浏览智能体的私有知识（别名：`/library`、`/codex`）。
- `/system` — 浏览智能体的系统文件（system.md、covenant 等）。
- `/daemons` — 检视各智能体的守护进程运行记录。
- `/notification` — 查看当前 Agent 最近收到的通知块，并可左右切换。
- `/presets` — 查看本智能体可用 `/refresh` 切换的预设。

### 网络与分享
- `/mailbox` — 浏览全部邮箱消息与附件。
- `/projects` — 浏览已注册的项目及其网络。
- `/export` — 导出可复用的配方以便分享。
- `/viz` — 在浏览器中打开网络可视化。

### 配置与诊断
- `/setup` — 智能体设置向导（提供商、模型、能力）。
- `/settings` — TUI 偏好（主题、页大小、语言）。
- `/mcp` — 外部服务桥接的 MCP 控制面板。
- `/doctor` — 诊断连接问题。
- `/login` — 查看与管理已保存的凭证。

### 本视图与退出
- `/help` — 打开本帮助指南。
- `/quit` — 退出 `lingtai-tui`（智能体仍在后台运行）。

## 命令详解

### `/btw` — 向智能体提旁问
**用法：** `/btw <你的问题>`

将问题作为一次*洞察询问*投递。智能体会反思并回答，而不打断它当前正在做的事——
让你在不让它脱轨的前提下探查其思路。适合询问与当前任务无关的题外话，又不想像普通
消息那样打断它的专注。

### `/insights` — 立即索取洞察
**用法：** `/insights`

让智能体立刻观察当前任务并给出 2–3 条具体观察。当你想立即、结构化地了解智能体所见，
而不愿等待自动洞察节奏（在 `/settings` 中开关）时使用。

### `/goal` — 引导创建 active goal
**用法：** `/goal` · `/goal <目标草稿>`

向当前 Agent 的 `.notification/system.json` 投递一个 `goal.request` 事件。
Agent 应读取 goal manual，和你一起明确 objective、success criteria、reminder
delay 与取消语义，并且只在你确认细节后才写入 `.notification/goal.json`。`/goal`
本身不会直接创建 goal 文件。之后 dismiss `goal.reminder` 只会隐藏那条提醒；真正取消
goal 需要删除 `.notification/goal.json`，或把其中 status 标为 inactive/cancelled/done。

### `/sleep` — 让智能体休眠
**用法：** `/sleep`（当前智能体）· `/sleep all`（项目内全部智能体）

休眠会在保留完整状态的前提下暂停智能体，之后可用 `/cpr` 唤醒。当你想停止智能体消耗
资源但打算原地恢复时使用。若需对操作系统进程进行更硬的冻结，请用 `/suspend`。

### `/suspend` — 挂起智能体
**用法：** `/suspend`（当前智能体）· `/suspend all`（项目内全部智能体）

完全冻结智能体进程，必须先 `/cpr` 复活才能恢复。当你需要对底层进程进行硬停止时使用——
比 `/sleep` 更重。若只想要可恢复的暂停，请用 `/sleep`。

### `/cpr` — 复活已挂起或已死亡的智能体
**用法：** `/cpr`（当前智能体）· `/cpr all`（项目内全部智能体）

让已挂起或已死亡的智能体重新运行。用于恢复被 `/sleep`、`/suspend` 暂停的智能体，或
抢救已死亡者。常规重启请优先用 `/refresh`——`/cpr` 专用于已宕机的智能体。

### `/refresh` — 硬重启智能体
**用法：** `/refresh`（重启，回到默认预设）· `/refresh <名称>`（切换预设，如
`/refresh mimo`）· `/refresh all`（全部智能体）

从磁盘重新加载 `init.json`、能力和全部配置，然后重启智能体。传入预设名会在重启前切换
当前预设；该预设须在智能体的 `manifest.preset.allowed` 列表中（见 `/presets`）。这是
改完配置后的首选命令，也用于切换不同的 LLM/能力预设——只有当智能体确已宕机时才用
`/cpr`。

### `/clear` — 清空上下文窗口并重启
**用法：** `/clear`

清空智能体的整个上下文窗口，并以空白对话重启。身份、便笺与 codex 都会保留——只有实时
对话被清除。当对话积累了噪声或偏离方向、你想要干净起点又不愿丢失持久记忆时使用。若想
先保存上下文，请改用 `/molt`。

### `/molt` — 立即让智能体蜕变
**用法：** `/molt`

强制智能体立即蜕变：先保存上下文，再重置对话窗口。当上下文窗口将满、又不想丢失智能体
已积累的内容时使用。与 `/clear` 不同，蜕变会在重置前保存上下文。

### `/nirvana` — 清除一切，从零开始
**用法：** `/nirvana`

删除 `.lingtai/` 及全部智能体数据，使项目回到全新状态。**⚠️ 不可逆**，无法撤销。若只想
要干净的对话，请用 `/clear`；若想在保留上下文的前提下重置，请用 `/molt`。只有当你确实
想让项目从零重来时才用 `/nirvana`。

### `/kanban` — 打开智能体网络面板
**用法：** `/kanban`

显示网络中每个智能体的面板：属性、LLM 配置、能力，以及上下文用量一览。当你想在单一终端
视图中比较所有智能体的配置及其上下文窗口的充盈程度时使用。若需图形化的浏览器视图，请用
`/viz`。

### `/skills` — 浏览技能目录
**用法：** `/skills`

打开技能目录——智能体可按需调用的可复用流程。`Ctrl+T` 在智能体间切换；对某技能按 `Enter`
可深入其文件夹中的文件。用于查看智能体能按需做什么，或检视某个具体技能。

### `/knowledge` — 浏览智能体的私有知识
**别名：** `/knowledge`、`/library`、`/codex` · **用法：** `/knowledge`

浏览智能体的私有知识——本地决策、笔记、参考和迁移而来的 codex 条目，包括其积累研究与
发现的持久知识库。`Ctrl+T` 在智能体间切换。用于阅读智能体为自己记录的内容，区别于共享的
技能目录（`/skills`）或系统文件（`/system`）。

### `/system` — 浏览智能体的系统文件
**用法：** `/system`

浏览智能体的系统文件：`system.md`、covenant、principle、procedures、pad 及 llm 配置。
`Ctrl+T` 在智能体间切换。用于阅读定义一个智能体的持久骨架——其治理文档与运行时配置——而
非其对话或私有知识（`/knowledge`）。

### `/daemons` — 浏览守护进程运行记录
**用法：** `/daemons`

打开守护进程浏览器：检视各智能体的守护进程运行及其状态、完整任务、完整 `chat_history`
交互，以及完整的工具/事件记录。用于追溯某个后台守护进程究竟做了什么。

### `/notification` — 查看最近通知块
**用法：** `/notification`

显示当前聚焦 Agent 最近收到的 agent-visible 通知块（从 `logs/log.sqlite` 重建）。当通知徽标看起来不对、Agent 说有通知，或你想检查最近实际注入给 Agent 的通知块时使用。左/右切换最近的通知块，`r` 重新加载，Esc/`q` 返回聊天；该视图只读，不会 dismiss 或修改通知。

### `/presets` — 打开预设库
**用法：** `/presets`

打开作用于当前智能体的预设库，仅显示该智能体 `manifest.preset.allowed` 列表中的预设——
即你能用 `/refresh <名称>` 切换的那些。当前激活的预设以 ● 标记。你可查看每个预设的 LLM
与能力，并为其打 1–5 星的成本/质量等级（越高越好）；标签会传播到智能体并指导守护/分身
的选择。本视图为只读检视加标签编辑——完整的预设创建在 `/setup` 中进行。

### `/mailbox` — 浏览全部邮箱消息
**用法：** `/mailbox`

浏览每一条邮箱消息——收件、发件与 IMAP——包含完整正文和内联附件渲染。当你想要某智能体
的完整邮件记录（含外部 IMAP 邮件与附件），而非实时对话流时使用。

### `/projects` — 浏览已注册的项目
**用法：** `/projects`

列出已注册的项目及其智能体网络。当你跨多个 LingTai 项目工作、想在一处查看它们及其网络
时使用。

### `/export` — 导出可复用配方
**用法：** `/export` · `/export recipe`（显式形式）

将当前配置导出为可复用的配方以便分享。用于把智能体配置打包复用与分享。

### `/viz` — 打开网络可视化
**用法：** `/viz`

在浏览器中打开智能体网络可视化——拓扑、邮件流与智能体状态。需要 `lingtai-portal` 位于
`PATH` 中。用于实时图形化查看网络中智能体如何关联与通信。若只需终端概览，请用 `/kanban`。

### `/setup` — 运行智能体设置向导
**用法：** `/setup`

打开智能体设置向导，可更改 LLM 提供商、模型、能力与运行时设置；更改会传播到所有智能体。
用于首次配置或对提供商、模型、能力的任何实质性更改。仅 TUI 偏好（主题、语言）请用
`/settings`；完整的预设创建也在此向导中进行。

### `/settings` — 编辑 TUI 偏好
**用法：** `/settings`

打开 TUI 偏好：主题、邮件页大小、语言，以及自动洞察开关。用于 TUI 自身的外观与客户端
行为。若要更改智能体的提供商、模型或能力，请用 `/setup`。

### `/mcp` — 打开 MCP 控制面板
**用法：** `/mcp`

打开 MCP 控制面板，查看各 MCP 桥接的资源、状态与配置——这些桥接（IMAP 邮件、Telegram、
飞书、微信）将智能体连接到外部服务。用于配置或排查外部服务集成。改动 MCP 配置后，运行
`/refresh` 使其生效。

### `/doctor` — 诊断连接问题
**用法：** `/doctor`

对智能体的连通性运行诊断——检查 API 密钥、模型可用性与网络配置——并报告结果。当智能体
无法启动或停止响应、你怀疑是凭证、模型或网络问题时使用。若 `/doctor` 指出凭证问题，可配合
`/login` 重新认证。

### `/login` — 查看与管理凭证
**用法：** `/login`

显示所有已保存凭证的认证状态，并允许在需要时重新认证。当某提供商凭证过期或需要重新登录
时使用——通常是为响应 `/doctor` 暴露的问题。

### `/help` — 打开本帮助指南
**用法：** `/help`

打开你正在阅读的帮助指南：斜杠命令工作方式的概览，以及每个命令的参考条目。按 `Esc` 或 `q`
返回邮件视图。任何时候想查阅某命令的作用或发现尚未用过的命令时使用。

### `/quit` — 退出 lingtai-tui
**用法：** `/quit`

退出 `lingtai-tui`。除非你先挂起了智能体，否则它们会在后台继续运行。若想让智能体也停止，
退出前请用 `/suspend all`（或 `/sleep all`）。
